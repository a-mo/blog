一个Spring注入的问题

2016-06-06
#spring

今天在码代码时遇到一个问题。
环境是在Spring+SringMVC+Mybatis。
在一个Service中通过注解注入另一个Service时，报了以下错误：

No unique bean of type [xxx] expected single matching bean but found 2

根据异常信息来看的话，是说通过Autowired注入对象失败，原因是本该找到一个类，结果却找到了两个类。究其原因的话是因为我要注入的这个类和另一个类都继承自同一个父类，而注释@Autowired是根据类型匹配，相应的解决办法应该是改用@Resource来注解，这个注解是按照名称来匹配类，同时在要注入的类上添加名称标识，如下：
//Service类1
public class Demo{
@Resource(name=”injection”)
private Injection injection;
}
//Service类2
@Service(value=”injection”)
public class Demo{
}

这样修改应该是需要把Autowired都改为Resource，继承自同一个父类的这两个类都该添加名称标识，奈何现在其中一个类已被封装，父类也被封装，不方便修改，只能重写了一个父类，使两个类不再出于同一父类。
PS：其实这个问题值得好好研究下Spring相应的文档和源码，我的另一个Person类也有类似的情况，也是与另一个类同样继承自一个父类，却不需要修改Autowired，值得深入探究下这个问题。
